# LLM-пайплайн суммаризации новостей

Скрипт читает новости из CSV-файла, отправляет каждую запись в LLM через
OpenAI-совместимый API, проверяет структуру ответа и сохраняет все результаты
в один JSON-файл.

## Что делает пайплайн

1. Загружает и проверяет `data/news.csv`.
2. Формирует отдельный запрос для каждой новости.
3. Получает от модели JSON с кратким содержанием, ключевыми фактами и
   категорией.
4. Проверяет обязательные поля и допустимые значения.
5. Сохраняет итог в `results/summaries.json`.

Для запуска достаточно Python 3.10 или новее. Внешние библиотеки не нужны.

## Быстрый запуск

```bash
git clone https://github.com/linsivs/llm-pipeline.git
cd llm-pipeline
python3 summarize_news.py
```

По умолчанию используется публичный OpenAI-совместимый endpoint Pollinations.
Для него API-ключ не требуется. Настройки можно скопировать в локальный файл:

```bash
cp .env.example .env
```

После успешного запуска программа выведет:

```text
Готово: 3 новостей сохранено в results/summaries.json
```

## Настройка другого LLM API

Скрипт совместим с API, которые поддерживают формат Chat Completions. Параметры
задаются переменными окружения или в файле `.env`:

```dotenv
LLM_API_URL=https://example.com/v1/chat/completions
LLM_MODEL=your-model-name
LLM_API_KEY=your-api-key
LLM_TIMEOUT=90
LLM_RETRIES=2
```

Файл `.env` исключён из Git, поэтому секретный ключ не попадёт в репозиторий.

## Формат входных данных

CSV должен содержать пять обязательных столбцов:

| Столбец | Описание |
|---|---|
| `id` | уникальный идентификатор новости |
| `published_at` | дата публикации |
| `title` | заголовок |
| `text` | текст для суммаризации |
| `source_url` | ссылка на источник |

Пример:

```csv
id,published_at,title,text,source_url
1,2025-04-14,"OpenAI представила семейство GPT-4.1","OpenAI выпустила через API модели GPT-4.1...","https://openai.com/index/gpt-4-1/"
```

В репозитории находится готовый входной файл с тремя новостями и ссылками на
официальные источники: [`data/news.csv`](data/news.csv).

## Формат результата

Модель должна вернуть объект следующего вида:

```json
{
  "summary": "Краткое содержание новости.",
  "key_points": [
    "Первый ключевой факт",
    "Второй ключевой факт"
  ],
  "category": "технологии"
}
```

Допустимые категории: `технологии`, `наука`, `бизнес`, `общество`, `другое`.
Скрипт отклоняет ответ, если JSON не разбирается или нарушает эту схему.

Фактический результат запуска сохранён в
[`results/summaries.json`](results/summaries.json). Фрагмент результата:

```json
{
  "id": "2",
  "title": "GitHub запустил бесплатный тариф Copilot",
  "summary": "GitHub объявил о запуске бесплатного тарифа Copilot, доступного в Visual Studio Code, позволяющего генерировать до 2000 дополнений кода и 50 чатов в месяц, с выбором моделей Claude 3.5 Sonnet или GPT-4o.",
  "category": "технологии"
}
```

Метаданные выходного файла также содержат дату генерации, фактическую модель и
статистику токенов, которую вернул API.

## Параметры командной строки

```bash
python3 summarize_news.py \
  --input data/news.csv \
  --output results/summaries.json \
  --limit 2
```

- `--input` — путь к входному CSV;
- `--output` — путь к выходному JSON;
- `--limit` — необязательное ограничение количества новостей.

## Проверка

Запуск автоматических тестов:

```bash
python3 -m unittest discover -s tests -v
```

Тесты проверяют чтение CSV, обработку некорректных данных, разбор JSON от LLM,
валидацию структуры и сохранение результата.
